General

Welcome to the cloudonix.io developers portal. This portal contains API documentation and additional information, provided as a tool to assist cloudonix.io customers to develop their own cloudonix.io plugins and business logic. Cloudonix.io follows a simple, straight forward, REST based API set - providing access to various elements of the platform to the developer.

Producers and Consumers

The platform includes various producer and consumer APIs. A consumer API indicates an API operating between an active application or process, with the customers billing or business logic. A producer API indicates an API meant primarily for initiating functions or manipulating data.
A classic consumer API shall be the /call API, initiated from the Cloudonix vapp server, to the customers external business logic.
A classic producer API shall be the /provision API, initiated from the customers business logic, to the Cloudonix provisioning API.

Error Handling

By default, all methods should return a 200/202 HTTP result code upon success, or a 4xx/5xx Error upon failure.
Errors should be formatted as following: 
{
  "error_code:" 502,
  "error_message": "Detailed error message"
}

Empty Responses

Some APIs may return an empty result, accompanied with a 202 response. A 202 response indicates that the request had completed successfully and no additional information is provided in respect to this query, eg. A delete request.
